Role Lead AI Engineer Domain Geospatial media planning Focus Reliable production agents Surface Conversational · Map-first
01

The brief

Planning out-of-home advertising means filtering a large national inventory across geography, audience, format, price, and proximity to real-world locations. Traditionally it's a slow, expert-led workflow: planners hand-build filters, cross-reference maps, and iterate by hand.

It doesn't scale to non-specialists, and results are hard to reproduce across sessions. The mandate was to make planning conversational — and to make it reliable enough to trust with real budgets.

02

What I built

A conversational AI agent embedded directly in the planning product.

A user describes what they want; the agent interprets it, runs structured geospatial searches, and returns synchronized map + table results. The conversation is stateful — users refine turn by turn while the map, table, and selection stay in lock-step.

Crucially, the model never touches data. It emits a structured specification that a deterministic layer validates and normalizes before anything executes.

03

How it works

A layered, provider-abstracted design. The model participates in a bounded, observable tool-use loop and proposes a structured spec; a deterministic guard layer validates and normalizes it; a query builder turns it into parameterized geospatial queries. Amber is where the model proposes; teal is where deterministic code verifies.

The model never queries data. It proposes a spec; deterministic code validates it, executes it, and returns the result to the client.

04

Engineering for trust

The hard part of production AI isn't the happy path — it's everything the model gets subtly wrong. Each known failure mode is caught by deterministic code and pinned by a regression test, so behavior can't silently drift.

non-determinism

Same request, same result

A normalization layer plus a golden-test suite make outputs reproducible instead of sampled.

state conflict

Live UI vs. spoken intent

On-screen filter state and a just-issued instruction are reconciled so neither silently overrides the other.

unit ambiguity

Metres, not millions

Natural-language quantities are disambiguated deterministically before they ever reach a query.

range hygiene

No impossible filters

"Under X" then "over X" can't collapse into a contradictory, empty result set.

answer-from-memory

Changes actually execute

A filter or display change re-runs the search rather than being recited from a prior turn.

human factors

Graceful escalation

Repeated dissatisfaction switches the agent from "retry" to "reset and hand off" instead of looping.

05

Key capabilities

  • +Natural language → structured query. Intent, attribute filters, and geospatial scope extracted from free text.
  • +Agentic tool-use loop. The model chooses among typed tools — search, aggregate, compare, export, corridor search — rather than following a fixed script.
  • +Proximity & composition targeting. Media near categories of real-world places, with format ratios, orientation, and physical spacing between assets.
  • +Stateful multi-turn refinement. Context carry-forward, broaden/narrow, and per-segment editing across a multi-location plan.
  • +Ambiguity handling. Asks a single, targeted clarifying question when a request is genuinely under-specified.
  • +Deterministic guardrails. A normalization layer repairs model and UX failure modes before any query runs.

07

Security & scale

Security posture
  • No model-authored data access — the model emits a validated spec; all access is parameterized server-side.
  • Encrypted, scoped credentials — provider keys never reach the client.
  • Role-based gating of privileged and diagnostic capabilities.
  • Provenance surfacing when data comes from a fallback provider, so results stay auditable.
Performance & scale
  • In-memory result reuse avoids redundant round-trips on refinement.
  • Cached external lookups stop refinements re-hitting paid providers or wobbling counts.
  • Single-pass bulk selection instead of walking pages.
  • Prompt/context caching & bounded loops to control per-turn latency and cost.
